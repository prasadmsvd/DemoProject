📚 Git Workflow Guide

Document Reference: Shared for team collaboration and best practices

1. Check Remote Repository Name

To view the remote repository linked to your project:

List all remotes:

git remote

List remotes with URLs:

git remote -v

Example Output:

origin https://github.com/prasadmsvd/DemoProject.git (fetch)
origin https://github.com/prasadmsvd/DemoProject.git (push)

2. Create New Branch & Switch To It

When starting a new feature or fix, create a dedicated branch:

Method 1 (Traditional):

git checkout -b <branch-name>

Method 2 (Git 2.23+):

git switch -c <branch-name>

Example:

git checkout -b feature/user-authentication
💡 Tip: Use descriptive branch names like feature/feature-name, bugfix/bug-name, or hotfix/issue-name

3. Push New Branch To Remote Repository

After making commits, push your branch to the remote repository:

Push with upstream tracking:

git push -u origin <branch-name>

Alternative:

git push --set-upstream origin <branch-name>

Example:

git push -u origin feature/user-authentication
ℹ️ Note: The -u flag sets upstream tracking. Future pushes only need git push

4. How To Raise a Pull Request (PR)

Step 1: Navigate to Remote Repository

Go to your repository on GitHub (or your Git hosting platform)

Step 2: Switch to Your Branch

Click on the branch dropdown and select your newly pushed branch

Step 3: Initiate Pull Request

You should see a prompt to "Compare & pull request". Click on it. If not visible, go to the "Pull requests" tab and click "New pull request"

Step 4: Select Base Branch

Choose the base branch (usually main or develop) where your changes will be merged

Step 5: Fill PR Details
  • Title: Clear, concise description of changes
  • Description: Detailed explanation of what was changed and why
  • Reviewers: Assign colleagues for code review
  • Labels: Add relevant labels (bug, feature, documentation, etc.)
Step 6: Create Pull Request

Click "Create Pull Request" button

Step 7: Review & Address Feedback

Wait for code review. Address any feedback by making additional commits to the same branch

Step 8: Merge PR

Once approved, click "Merge pull request" and delete the branch after merging

5. Additional Useful Git Commands

View current branch:

git branch -v

List all branches (local and remote):

git branch -a

Switch to existing branch:

git checkout <branch-name>

Check git status:

git status

Stage changes:

git add .

Commit changes:

git commit -m "Your commit message"

Push commits:

git push

Pull latest changes:

git pull

6. Best Practices

  • ✅ Always create a new branch for features/fixes
  • ✅ Use descriptive branch names
  • ✅ Make small, focused commits with clear messages
  • ✅ Push regularly to avoid losing work
  • ✅ Keep commits atomic (one logical change per commit)
  • ✅ Write meaningful PR descriptions
  • ✅ Request code review before merging
  • ✅ Delete branches after merging
  • ✅ Keep your branch updated with the base branch
